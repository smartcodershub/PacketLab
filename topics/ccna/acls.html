Access Control Lists

Filter traffic with standard and extended ACLs — permit/deny rules and placement

What are ACLs?

ACLs are ordered lists of permit/deny rules applied to router interfaces. They filter traffic based on source IP, destination IP, protocol and port numbers.

ACL Processing Rules

  • Rules are checked top to bottom — first match wins
  • There is an implicit deny all at the end of every ACL
  • Standard ACL → place close to the destination
  • Extended ACL → place close to the source

Standard ACL (1–99)

Filters only on source IP address.

Router(config)# access-list 10 permit 192.168.1.0 0.0.0.255
Router(config)# access-list 10 deny any

Router(config)# interface gi0/1
Router(config-if)# ip access-group 10 out

Extended ACL (100–199)

Filters on source IP, destination IP, protocol, and port.

! Allow HTTP from 192.168.1.0/24 to any
Router(config)# access-list 100 permit tcp 192.168.1.0 0.0.0.255 any eq 80

! Allow DNS
Router(config)# access-list 100 permit udp any any eq 53

! Deny everything else (implicit, but good practice to see)
Router(config)# access-list 100 deny ip any any

Router(config)# interface gi0/0
Router(config-if)# ip access-group 100 in

! Verify
Router# show access-lists
Router# show ip interface gi0/0
