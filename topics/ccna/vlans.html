VLANs & Trunking

Virtual LANs, 802.1Q trunk links, and inter-VLAN routing

What is a VLAN?

A VLAN (Virtual LAN) logically segments a switch into multiple broadcast domains without physical separation. Devices in different VLANs cannot communicate directly — they need a router or Layer 3 switch.

Benefits

  • Reduces broadcast traffic
  • Improves security by isolating departments
  • Simplifies network management

VLAN Configuration (Cisco IOS)

! Create VLANs
Switch(config)# vlan 10
Switch(config-vlan)# name SALES
Switch(config)# vlan 20
Switch(config-vlan)# name HR

! Assign port to VLAN (access port)
Switch(config)# interface fa0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10

! Verify
Switch# show vlan brief

Trunk Links (802.1Q)

Trunk ports carry multiple VLANs between switches using 4-byte VLAN tags inserted into frames.

Switch(config)# interface gi0/1
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk encapsulation dot1q
Switch(config-if)# switchport trunk allowed vlan 10,20,30

Switch# show interfaces trunk

Inter-VLAN Routing (Router-on-a-Stick)

Router(config)# interface gi0/0.10
Router(config-subif)# encapsulation dot1q 10
Router(config-subif)# ip address 192.168.10.1 255.255.255.0

Router(config)# interface gi0/0.20
Router(config-subif)# encapsulation dot1q 20
Router(config-subif)# ip address 192.168.20.1 255.255.255.0
